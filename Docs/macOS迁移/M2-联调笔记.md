## OpenAI-compatible 冒烟

脚本：`Scripts/m2-openai-smoke.csx`
配置：`~/.tianming/smoke.json`（本机不入库）

### 跑法

```bash
# 首次：装 dotnet-script
dotnet tool install -g dotnet-script

# 准备配置
mkdir -p ~/.tianming
cat > ~/.tianming/smoke.json <<'EOF'
{
  "BaseUrl": "https://api.deepseek.com",
  "ApiKey":  "REPLACE-WITH-YOUR-KEY",
  "Model":   "deepseek-chat"
}
EOF
# 编辑 ~/.tianming/smoke.json 填入真实 key

# 前置：先 build Tianming.AI（脚本里 #r 引用 bin/Debug/net8.0/...dll）
dotnet build src/Tianming.AI/Tianming.AI.csproj --nologo

# 跑
dotnet script Scripts/m2-openai-smoke.csx
```

### 常见问题

- 401 鉴权失败：确认 Bearer header 写法与 `ApiKey` 值是否完整
- 404：endpoint 路径是否以 `/v1/chat/completions` 为后缀？`OpenAICompatibleChatClient` 会自动归一化 BaseUrl
- SSE 卡顿：看 `Transfer-Encoding: chunked` 是否被中间代理剥掉
- 本机 NuGet 若对某些 `Microsoft.ML.OnnxRuntime` 版本挂起，设 `HTTP_PROXY`/`HTTPS_PROXY` 或换已 cached 的版本（本仓 M2 用 1.24.2）

## ONNX 真向量

放模型文件到：`~/Library/Application Support/Tianming/Models/`

推荐模型（手动下载）：
- `bge-small-zh-v1.5`：~90MB，512 维，中文友好
  - `pytorch_model.bin` → 用 transformers 导出成 onnx；或 HuggingFace 上搜已转好的版本
  - `vocab.txt` 必需

```csharp
using TM.Services.Framework.AI.SemanticKernel;

var settings = EmbeddingSettings.Default with
{
    ModelFilePath = "/Users/<user>/Library/Application Support/Tianming/Models/model.onnx",
    VocabFilePath = "/Users/<user>/Library/Application Support/Tianming/Models/vocab.txt",
    MaxSequenceLength = 512
};
using var embedder = OnnxEmbedderFactory.Create(settings);
var v = embedder.Embed("今天天气不错。");
Console.WriteLine($"Dim={v.Length}  NormSq={v.Sum(x=>x*x):F4}");   // NormSq 应≈1.0
```

## plan 偏离说明

- 原 plan 指定 `Microsoft.ML.OnnxRuntime 1.18.1`；实际用 **1.24.2**。
  原因：本机 NuGet 从 1.18.1 下载后在签名校验阶段稳定挂起（>5 分钟，0% CPU，即使加 `HTTP_PROXY=127.0.0.1:1082` 通过系统代理也不解），而 1.24.2 已本地 cached 且 `InferenceSession` API 在 1.18→1.24 期间未变更（保持 `new InferenceSession(path)`、`session.Run(inputs)`、`outputs.First().AsTensor<float>()`）。M2 plan 原文也允许"同 major 稳定版即可"的弹性，本次偏离不影响 API 与降级逻辑。
- `OnnxTextEmbedder.Embed` 里 `DenseTensor<long>` 的 shape 参数类型为 `ReadOnlySpan<int>`（不是 `long`），故代码用 `new[] { 1, inputIds.Length }` 而非 plan 原稿的 `new[] { 1L, inputIds.Length }`。这是运行时类型约束，非设计变动。
