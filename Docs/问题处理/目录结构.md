## 四层架构说明

依赖方向：**Modules → Services → Framework ← Core**

### Core（核心层）

`Core/App/` — 应用入口、主窗口、DI 容器配置、生命周期管理。

### Framework（框架层）

`Framework/` — 不依赖其他层，提供所有可复用基础设施：

| 子目录 | 职责 |
|--------|------|
| **Common/** | 通用控件、转换器、辅助类、样式、ViewModel 基类 |
| **Appearance/** | 主题管理、智能配色、动画、自动主题、字体 |
| **UI/** | 窗口框架、工作区三栏布局（左导航/中编辑/右对话） |
| **User/** | 用户账号、登录、OAuth、密码安全、偏好设置 |
| **Notifications/** | 通知管理、音效、系统通知集成 |
| **SystemSettings/** | 代理、日志、数据清理、系统信息 |

### Services（服务层）

`Services/` — 跨模块共享的公共服务：

| 子目录 | 职责 |
|--------|------|
| **Framework/AI/** | Semantic Kernel 集成、API Key 轮换、用量监控、查询路由 |
| **Framework/EntityManager/** | 实体管理 |
| **Framework/Notification/** | 全局通知与免打扰 |
| **Framework/Settings/** | 全局设置管理 |
| **Framework/SystemIntegration/** | 托盘图标、Windows 通知 |
| **Modules/ProjectData/** | 项目数据、索引、发布、变更检测 |
| **Modules/VersionTracking/** | 版本追踪 |

### Modules（模块层）

`Modules/` — 四大业务模块：

| 模块 | 职责 |
|------|------|
| **AIAssistant/** | 模型管理、提示词、版本测试、记忆管理 |
| **Design/** | 世界观/角色/势力/地点/剧情规则、拆书、素材库 |
| **Generate/** | 大纲、卷设计、蓝图、章节生成 |
| **Validate/** | 多维校验、一致性检查、章节修复 |

## 服务放置决策

```
新建 Service →  被多个模块共享？
                ├─ 是 → Services/Framework/ 或 Services/Modules/
                └─ 否 → Framework/[功能]/Services/ 或 [子功能]/ 内
```

## 组件使用指南

### 标准样式（自动生效）

`App.xaml` → `AllStyles.xaml` → `ImplicitStyles.xaml`，所有控件自动继承隐式样式。

### StandardDialog

```csharp
StandardDialog.ShowInfo("操作成功", "提示");
StandardDialog.ShowError("操作失败", "错误");
StandardDialog.ShowConfirm("确定删除？", "确认");
```

### GlobalToast

```csharp
GlobalToast.Success("保存成功", "配置已保存");
GlobalToast.Error("保存失败", "文件不存在");
```

### StoragePathHelper

```csharp
var path = StoragePathHelper.GetFilePath("Framework", "Appearance/ThemeManagement", "themes.json");
// → Storage/Framework/Appearance/ThemeManagement/themes.json
```

**Storage 路径必须对应代码层级结构，禁止手动拼接路径。**

## 开发规范

### 命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 命名空间 | PascalCase | `TM.Framework.Common.Controls` |
| 类/方法/属性 | PascalCase | `StandardDialog` / `SwitchTheme()` |
| 接口 | I + PascalCase | `IAIChatService` |
| 私有字段 | _camelCase | `_themeManager` |
| 参数 | camelCase | `themeType` |

### 编码

- 颜色资源必须用 `DynamicResource`，禁止硬编码
- 弹窗必须用 `StandardDialog`，禁止 `MessageBox`
- 日志必须用 `TM.App.Log()`，禁止 `Console.WriteLine`
- 存储路径必须用 `StoragePathHelper`，禁止手动拼接

## 常见问题

**Q: 新 Service 放哪？**
A: 被多个模块共享 → `Services/`；只被单个功能用 → `Framework/[功能]/Services/`

**Q: Storage 路径怎么组织？**
A: 对应代码层级：`Framework/Appearance/ThemeManagement/` → `Storage/Framework/Appearance/ThemeManagement/`

**Q: 页面显示"功能尚未开发"？**
A: 检查 View 文件是否存在、路由映射是否添加（`RouteConfig.cs`）、类名是否含完整命名空间。
