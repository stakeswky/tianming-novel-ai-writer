## 2. DataTreeView - 数据内容树（★ v2.3强化）

**文件位置**: `Framework/Common/Controls/DataTree/DataTreeView.xaml`

标准化的数据列表/树展示组件，支持5层递归树形结构和智能操作按钮集成。

### 功能特性（v3.3 增强）
- ✅ **5层递归支持**（原2层 → 5层）
- ✅ **智能操作按钮组**（新建、删除、保存、全部删除）
- ✅ **一键启用/禁用（BulkToggle）**：支持对选中主分类（含子分类）或全量分类下数据执行批量启用/禁用
- ✅ **集合变化实时响应**（CollectionChanged事件订阅）
- ✅ **双击选择机制**（单击展开、双击选择）
- ✅ **顶级选项自动注入**（可配置"主页导航"等顶级节点）
- ✅ **层级限制控制**（MaxLevel 1-5可配置）
- ✅ **按钮可用性联动**：新增 `IsAddEnabled` / `IsDeleteEnabled`，可由宿主布局统一控制
- ✅ **操作回调管线**：`AfterActionCommand` 在新建/保存/删除后统一通知 ViewModel
- ✅ **全部删除命令**：`DeleteAllCategoriesCommand` 自动处理二次确认 + 空集合提示
- ✅ **状态提示优化**：无条目时「全部删除」按钮自动显示 `暂无条目`，并使用标准确认文案
- ✅ **展开状态保持**：刷新树时自动保存/恢复节点展开状态（配合基类）
- ✅ 统一的列表/树形展示样式、悬停/选中视觉
- ✅ 彩色emoji图标支持

### 使用示例

#### 基础用法

```xml
xmlns:tree="clr-namespace:TM.Framework.Common.Controls"

<!-- 扁平列表模式（父节点可点击） -->
<tree:DataTreeView ItemsSource="{Binding RuleTree}" 
                   ParentClickMode="Select"
                   ParentNodeClickCommand="{Binding SelectRuleCommand}"/>

<!-- 树形结构模式（父节点展开/折叠，子节点可点击） -->
<tree:DataTreeView ItemsSource="{Binding CategoryTree}" 
                   ParentClickMode="Toggle"
                   ChildNodeClickCommand="{Binding SelectCategoryCommand}"/>
```

#### v2.3 高级用法（分类管理标准方案）

```xml
<!-- 左侧导航树（带智能按钮组） -->
<tree:DataTreeView 
    ItemsSource="{Binding CategoryDataTree}"
    ParentClickMode="Toggle"
    ShowActionButtons="True"
    MaxLevel="5"
    AddCategoryCommand="{Binding AddCategoryFromTreeCommand}"
    SaveCategoryCommand="{Binding SaveCategoryCommand}"
    DeleteCategoryCommand="{Binding DeleteCategoryFromTreeCommand}"/>

<!-- 父分类下拉选择（带顶级选项+双击选择） -->
<ComboBox Style="{StaticResource TreeComboBoxStyle}"
          ItemsSource="{Binding ParentCategoryTree}"
          IsDropDownOpen="{Binding IsTreeComboBoxOpen}"
          helpers:ComboBoxHelper.SelectedPath="{Binding SelectedParentPath}"
          helpers:ComboBoxHelper.MaxLevel="4"
          helpers:ComboBoxHelper.NodeDoubleClickCommand="{Binding SelectParentCategoryCommand}"/>
```

> ℹ️ **v3.3 提示**：建议同时绑定 `DeleteAllCategoriesCommand` 与 `AfterActionCommand`（通过 `TwoColumnEditorLayout.TreeAfterActionCommand` 透传），以便内容模块在新增、保存、删除、清空后立即刷新右侧视图。

#### ViewModel构建数据

```csharp
using TM.Framework.Common.Controls;

public ObservableCollection<TreeNodeItem> RuleTree { get; } = new();

private void BuildRuleTree()
{
    RuleTree.Clear();
    foreach (var rule in Rules)
    {
        RuleTree.Add(new TreeNodeItem
        {
            Name = rule.Name,
            Icon = rule.IsEnabled ? "✅" : "⚪",
            Tag = rule,
            IsExpanded = false,
            ShowChildCount = false
        });
    }
}
```

#### 多层树形结构

```csharp
private void BuildCategoryTree()
{
    CategoryTree.Clear();
    foreach (var category in MainCategories)
    {
        var parentNode = new TreeNodeItem
        {
            Name = category.Name,
            Icon = category.Icon,
            IsExpanded = true,
            ShowChildCount = true,
            Tag = category
        };
        
        foreach (var subCategory in category.SubCategories)
        {
            parentNode.Children.Add(new TreeNodeItem
            {
                Name = subCategory.Name,
                Icon = "📄",
                Tag = subCategory
            });
        }
        
        CategoryTree.Add(parentNode);
    }
}
```

## 10. DualTabContainer - 动态Tab容器

**文件位置**: `Framework/Common/Controls/Layout/DualTabContainer.xaml`

标准化的Tab页容器组件，支持XAML直接定义Tab内容，自动适配Tab数量。

### 功能特性
- ✅ **XAML直接定义** - 使用 `<local:TabItemData>` 标签定义Tab
- ✅ **动态数量适配** - 支持2个、3个、任意数量的Tab
- ✅ **圆角现代风格** - 内置ModernTabButtonStyle样式
- ✅ **自动单选逻辑** - 内置IsSelected状态管理
- ✅ **Emoji图标支持** - 每个Tab支持独立图标
- ✅ **统一间距管理** - ContentPadding默认10px
- ✅ **ContentProperty特性** - 简化XAML写法

### 使用示例

#### XAML引用

```xml
xmlns:common="clr-namespace:TM.Framework.Common.Controls.Layout"
xmlns:local="clr-namespace:TM.Framework.Common.Controls.Layout"

<!-- 外层容器需要Padding="0"，由组件统一管理间距 -->
<common:IconHeaderGroupBox Icon="📋" Title="详情" Padding="0">
    <common:DualTabContainer>
        <!-- Tab 1 -->
        <local:TabItemData Header="基本信息" Icon="📄" IsSelected="True">
            <StackPanel>
                <!-- Tab1内容 -->
            </StackPanel>
        </local:TabItemData>
        
        <!-- Tab 2 -->
        <local:TabItemData Header="高级配置" Icon="⚙️">
            <ScrollViewer>
                <!-- Tab2内容 -->
            </ScrollViewer>
        </local:TabItemData>
        
        <!-- Tab 3 (可选) -->
        <local:TabItemData Header="统计报告" Icon="📊">
            <Border>
                <!-- Tab3内容 -->
            </Border>
        </local:TabItemData>
    </common:DualTabContainer>
</common:IconHeaderGroupBox>
```

### 技术说明

#### 内部实现
- 使用 **RadioButton** 模拟Tab按钮（GroupName自动管理）


## FunctionalDetailForm - 自适应详情表单

**文件位置**: `Framework/Common/Controls/Forms/FunctionalDetailForm.xaml`

右栏详情区域的标准化容器组件，基于声明式字段定义自动生成自适应网格布局，并支持附加自定义内容。配合 `FunctionalDetailField`（`Framework/Common/Controls/Forms/FunctionalDetailField.cs`）统一管理字段分组、最小/最大宽度和权重，实现“自动吃掉”剩余空间的现代表单体验。

### 功能特性
- ✅ **自适应布局引擎**：根据 `MinWidth`、`MaxWidth`、`AllowGrow`、`GrowWeight` 自动安排列宽
- ✅ **分组与合并**：通过 `GroupKey` + `AllowMerge` 将同组字段排在同一行，并在空间允许时自动合并
- ✅ **标准标签区**：继承项目统一标签样式，确保与旧界面视觉一致
- ✅ **附加内容槽位**：`AdditionalContent` 用于放置长文本、卡片式滑块等扩展区域
- ✅ **统一文本编辑右键菜单**：内部自动为名称字段、图标 Fallback 等 TextBox 启用标准 3 项右键菜单（剪切/复制/粘贴）
- ✅ **遗留字段兼容**：保留 `LegacyNameContent` / `LegacyCategoryContent` / `LegacyTypeContent` 以兼容旧写法
- ✅ **布局日志**：自动记录异常布局场景，便于排查字段配置问题

### 字段定义（FunctionalDetailField）

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| Label | string | "" | 字段显示标签（使用标准样式，不可省略） |
| GroupKey | string | "default" | 分组键，同行字段需使用相同值 |
| MinWidth | double | 200 | 最小宽度，布局按此值起算 |
| MaxWidth | double | double.PositiveInfinity | 最大宽度，上限0表示不限制 |
| AllowGrow | bool | true | 是否允许根据剩余空间扩展 |
| GrowWeight | double | 1 | 扩展权重，值越高优先分配剩余空间 |
| AllowMerge | bool | true | 是否允许与同组字段合并行 |
| Order | int | 0 | 排序权重，值小的先布局 |
| Content | UIElement | null | 字段内容，通常为 TextBox / ComboBox |

> 📌 提示：常规字段建议 `MinWidth` 180-220，关键字段（如名称）可设置 `GrowWeight="2"` 以吸收剩余空间。

> ⚠️ **分类数据初始化规范（强制）**
>
> - 所有使用 `TwoColumnEditorLayout` 的功能，必须在 `Storage/Modules/[模块]/[子功能]/` 下建立 `*_categories_template.json` 模板。
> - ViewModel/Service 在启动时调用 `EnsureCategoriesSeeded()`，当正式数据不存在时从模板 JSON 复制，严禁在 C# 中硬编码种子列表。
> - 模板必须包含稳定的 `Id`、`ParentId`、`Level`、`Order` 字段，后续调整统一修改模板文件。
> - 推荐命名示例：`Storage/Modules/Creation/Materials/Analysis/analysis_categories_template.json`。
> - 服务层示例：
> ```csharp
> public void EnsureCategoriesSeeded()
> {
>     if (GetAllCategories().Any()) return;
>     var templatePath = StoragePathHelper.GetFilePath("Modules", "Creation/Materials/Analysis", "analysis_categories_template.json");
>     var templateJson = File.ReadAllText(templatePath);
>     var categories = JsonSerializer.Deserialize<List<AnalysisCategory>>(templateJson) ?? new();
>     File.WriteAllText(_categoriesPath, JsonSerializer.Serialize(categories, new JsonSerializerOptions { WriteIndented = true }));
> }
> ```
> - 当实现「全部删除」时，只清空正式数据文件，模板文件必须保留以便再次初始化。

## 15. AnimatedGradientStripe - 动态渐变侧边条

**文件位置**: `Framework/Common/Controls/Animations/AnimatedGradientStripe.xaml`

用于在树节点、列表项等场景提供连续流动的渐变高亮条，可在不同状态之间切换主题色，突出当前选中或父节点路径。

### 功能特性
- ✅ 提供 `Orientation`、`Thickness`、`StripeCornerRadius` 等参数控制外观
- ✅ 通过 `BaseBrush` 与 `PulseBrush` 组合实现平滑脉冲动画
- ✅ `IsActive` 控制动画启停（内部自动创建/释放 `Storyboard`）
- ✅ 支持自定义动画周期 `AnimationDuration`

### 使用示例

```xml
xmlns:animations="clr-namespace:TM.Framework.Common.Controls.Animations"

<animations:AnimatedGradientStripe Thickness="5"
                                   Orientation="Vertical"
                                   StripeCornerRadius="6,0,0,6"
                                   BaseBrush="{DynamicResource PrimaryColor}"
                                   PulseBrush="{StaticResource TreeHighlightGreenPulse}"
                                   IsActive="{Binding IsSelectionFocus}" />
```

> ℹ️ 建议与 `DataTreeView` 自定义节点模板搭配，用于高亮当前选中节点（绿色）或父节点路径（蓝色）。

### 主要依赖属性

| 属性 | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| Orientation | Orientation | Vertical | 动画方向（Vertical/Horizontal） |
| Thickness | double | 4 | 高亮条宽度 |
| BaseBrush | Brush | PrimaryColor | 基础渐变主色 |
| PulseBrush | Brush | SuccessColor | 脉冲颜色 |
| IsActive | bool | False | 是否启动动画 |
| AnimationDuration | TimeSpan | 0:0:2 | 动画循环周期 |

## 16. BusyOverlay - 全局忙碌遮罩

**文件位置**: `Framework/Common/Controls/Feedback/BusyOverlay.xaml`

提供覆盖式半透明遮罩与中心加载指示器，适用于 AI 生成等需要阻止用户操作的场景，推荐作为叠加层使用以保持宿主布局结构不变。

### 功能特性
- ✅ `IsBusy` 控制遮罩显示，并自动阻断命中测试
- ✅ 默认含 `LoadingIndicator`，支持 `OverlayContent` 自定义内容
- ✅ `OverlayBackground` 提供半透明灰层，可依据主题覆盖
- ✅ `OverlayCornerRadius` 让遮罩与宿主容器圆角对齐
- ✅ 作为 Grid 末子项叠加，保留原有内容与命名空间

### 使用示例

```xml
xmlns:feedback="clr-namespace:TM.Framework.Common.Controls.Feedback"

<Grid>
    <!-- 原窗口内容 -->
    <Border CornerRadius="4">
        <!-- ... -->
    </Border>

    <feedback:BusyOverlay IsBusy="{Binding IsAIGenerating}"
                          OverlayCornerRadius="4"
                          Panel.ZIndex="1"/>
</Grid>
```

### 主要依赖属性

| 属性 | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| IsBusy | bool | False | 控制遮罩显隐及命中测试 |
| OverlayBackground | Brush | #55000000 | 遮罩背景色 |
| OverlayContent | object | null | 自定义忙碌内容（默认 LoadingIndicator） |
| OverlayContentTemplate | DataTemplate | null | 自定义内容模板 |
| OverlayCornerRadius | CornerRadius | 0 | 遮罩圆角，建议与宿主容器一致 |

> ✅ 推荐与 `WritingWindowViewModel.IsAIGenerating` 等状态属性绑定，实现统一的全局忙碌体验。
