## 1. StoragePathHelper - 路径标准化工具

**文件位置**: `Framework/Common/Helpers/Storage/StoragePathHelper.cs`

**【强制使用】** 所有数据存储路径必须使用此工具类。

### 功能特性
- ✅ 自动适配开发/打包环境
- ✅ 自动创建目录结构
- ✅ 跨平台路径分隔符
- ✅ 详细日志输出

### 使用示例

```csharp
// 已全局引用，可直接使用

// 1. Storage目录下的数据文件
string filePath = StoragePathHelper.GetFilePath(
    "Framework",  // 层级：Framework/Services/Modules
    "Appearance/ThemeManagement/Themes",  // 子路径
    "dark_theme.json"  // 文件名
);

// 2. Framework目录下的资源文件
string themePath = StoragePathHelper.GetFrameworkPath(
    "Appearance/ThemeManagement/Themes"
);

// 3. 获取项目根目录
string projectRoot = StoragePathHelper.GetProjectRoot();
```

### API参考

```csharp
public static string GetFilePath(string layer, string subPath, string fileName)
public static string GetFrameworkPath(string subPath)
public static string GetProjectRoot()
public static string GetStorageRoot()
```

## 3.4 TextInputContextMenuHelper - 文本编辑右键菜单

**文件位置**: `Framework/Common/Helpers/UI/TextInputContextMenuHelper.cs`

为文本输入控件（TextBox / PasswordBox 等）提供统一的**剪切 / 复制 / 粘贴**三项右键菜单，使用与文件操作菜单一致的现代圆角样式。

### 功能特性
- ✅ 统一 3 项菜单：剪切(T) / 复制(C) / 粘贴(V)
- ✅ 支持 TextBox / RichTextBox（TextBoxBase）
- ✅ 支持 PasswordBox 等其他输入控件
- ✅ 自动应用圆角+阴影的标准 ContextMenu 样式
- ✅ 已通过隐式样式和双栏布局集成，大部分场景无需手动配置

### 使用示例（手动启用场景）

```xml
xmlns:helpers="clr-namespace:TM.Framework.Common.Helpers.UI"

<!-- 在自定义输入控件内部显式启用 -->
<TextBox
    Text="{Binding Value}"
    helpers:TextInputContextMenuHelper.EnableStandardEditMenu="True"/>

<PasswordBox
    helpers:TextInputContextMenuHelper.EnableStandardEditMenu="True"/>
```

> ℹ️ **提示**：
> - 普通 TextBox / PasswordBox 已通过 `ImplicitStyles.xaml` 隐式样式自动启用，无需在业务 View 中逐一设置。
> - `TwoColumnEditorLayout` 会在 Loaded 时对右栏所有 TextBoxBase 再做一次统一启用，确保方案 2 右栏输入区行为一致。
> - 仅在自定义控件内部（例如特殊组合控件）需要显式调用此 Helper。

## 3.5 RefreshEffectHelper - 刷新视觉反馈

**文件位置**: `Framework/Common/Helpers/UI/RefreshEffectHelper.cs`

为带有“刷新”菜单项的右键菜单提供统一的**轻量刷新视觉效果**。

### 功能特性
- 通过附加属性启用：`helpers:RefreshEffectHelper.EnableRefreshEffect="True"`
- 自动在 `ContextMenu.Opened` 时扫描菜单项
- 仅对 `Header` 含“刷新”的菜单项挂接效果
- 点击“刷新”时，对 `ContextMenu.PlacementTarget` 播放一次轻量闪烁动画（模拟 Windows F5 刷新效果）
- 默认已通过 `ImplicitStyles.xaml` 集成到全局 ContextMenu 隐式样式

### 使用示例（仅在自定义 ContextMenu 时需要显式启用）

```xml
xmlns:helpers="clr-namespace:TM.Framework.Common.Helpers.UI"

<ContextMenu helpers:RefreshEffectHelper.EnableRefreshEffect="True">
    <MenuItem Header="刷新" Command="{Binding RefreshCommand}"/>
    <MenuItem Header="删除" Command="{Binding DeleteCommand}"/>
</ContextMenu>
```

> ℹ️ **提示**：
> - 默认 ContextMenu 隐式样式已启用刷新效果，大多数右键菜单无需手动配置。
> - 仅在完全自定义的 ContextMenu（不使用隐式样式或需要显式控制启用/禁用）时，才需要手动设置附加属性。

## 6. AI相关辅助类

**文件位置**: `Framework/Common/Helpers/AI/`

### PromptGenerationService - 通用提示词生成服务

基于统一的元提示词模板，为各功能模块自动生成功能提示词（System Prompt）。

- 使用 JSON 协议返回结构化结果：`name`、`content`、`description`、`tags`。
- 统一替代旧的 `MetaPromptService` / `IPromptGenerator` / `PromptContext` 等接口。
- 所有需要自动生成 Prompt 的模块应依赖 `IPromptGenerationService`。

```csharp
var context = new PromptGenerationContext
{
    ModuleKey = "PromptTools.Management",
    ModuleDisplayName = "提示词管理",
    TemplateName = "角色设定模板",
    ExtraRequirement = "偏轻小说风格，突出人物性格",
    FieldNames = new[] { "Prompt正文", "说明" }
};

var service = new PromptGenerationService();
var result = await service.GenerateModulePromptAsync(context);

if (result.Success && !string.IsNullOrWhiteSpace(result.Content))
{
    // result.Content: 生成的 System Prompt 正文
    // result.Name / result.Description / result.Tags: 可选的结构化元信息
}
```
